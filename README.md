### 第02章：创建型模式

创建型模式主要关注对象的创建过程，将对象的创建与使用分离。

#### 2.1 单例模式（Singleton Pattern）
- 饿汉式、懒汉式、双重检查锁、静态内部类、枚举
- 单例的线程安全问题
- 单例的破坏与保护（序列化、反射）

#### 2.2 工厂方法模式（Factory Method Pattern）
- 简单工厂
- 工厂方法
- 适用场景：日志记录器、数据库访问

#### 2.3 抽象工厂模式（Abstract Factory Pattern）
- 产品族与产品等级结构
- 与工厂方法的区别
- 适用场景：跨平台 UI 组件

#### 2.4 建造者模式（Builder Pattern）
- 链式调用
- 与构造函数的区别
- 适用场景：复杂对象的构建（如 StringBuilder、Lombok @Builder）

#### 2.5 原型模式（Prototype Pattern）
- 浅克隆与深克隆
- Cloneable 接口
- 适用场景：对象复制、缓存优化

---

### 第03章：结构型模式

结构型模式关注类与对象的组合，通过继承或组合构建更大的结构。

#### 3.1 适配器模式（Adapter Pattern）
- 类适配器 vs 对象适配器
- 适用场景：接口不兼容问题、电源适配器

#### 3.2 桥接模式（Bridge Pattern）
- 抽象部分与实现部分分离
- 适用场景：多维度变化（如不同品牌、不同颜色的手机）

#### 3.3 组合模式（Composite Pattern）
- 树形结构
- 透明方式 vs 安全方式
- 适用场景：文件系统、菜单系统

#### 3.4 装饰器模式（Decorator Pattern）
- 动态扩展功能
- 与继承的区别
- 适用场景：IO 流（BufferedInputStream）、Java EE 中的装饰器

#### 3.5 外观模式（Facade Pattern）
- 统一接口封装
- 适用场景：复杂子系统简化调用（如 Spring JDBC）

#### 3.6 享元模式（Flyweight Pattern）
- 对象共享
- 内部状态 vs 外部状态
- 适用场景：连接池、线程池、字符串常量池

#### 3.7 代理模式（Proxy Pattern）
- 静态代理 vs 动态代理（JDK 动态代理、CGLIB）
- 适用场景：AOP、延迟加载、权限控制

---

### 第04章：行为型模式

行为型模式关注对象之间的通信、职责分配和算法的封装。

#### 4.1 责任链模式（Chain of Responsibility Pattern）
- 请求的传递与处理
- 适用场景：过滤器链（Filter Chain）、拦截器

#### 4.2 命令模式（Command Pattern）
- 请求封装为对象
- 适用场景：事务回滚、任务队列、撤销操作

#### 4.3 解释器模式（Interpreter Pattern）
- 语法解析
- 适用场景：表达式计算、SQL 解析

#### 4.4 迭代器模式（Iterator Pattern）
- 统一遍历接口
- 适用场景：Java 集合框架中的 Iterator

#### 4.5 中介者模式（Mediator Pattern）
- 对象间解耦
- 适用场景：MVC 中的 Controller、聊天室

#### 4.6 备忘录模式（Memento Pattern）
- 状态保存与恢复
- 适用场景：游戏存档、撤销操作

#### 4.7 观察者模式（Observer Pattern）
- 发布-订阅机制
- 适用场景：事件监听（EventListener）、消息队列

#### 4.8 状态模式（State Pattern）
- 状态驱动的行为变化
- 适用场景：订单状态、工作流引擎

#### 4.9 策略模式（Strategy Pattern）
- 算法封装与切换
- 适用场景：支付方式选择、压缩算法

#### 4.10 模板方法模式（Template Method Pattern）
- 算法骨架 + 具体实现
- 适用场景：JdbcTemplate、工作流定义

#### 4.11 访问者模式（Visitor Pattern）
- 数据结构与操作分离
- 适用场景：语法树遍历、文档处理
